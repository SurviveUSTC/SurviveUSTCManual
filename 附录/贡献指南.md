# 贡献指南

欢迎成为本手册的贡献者。本项目采用开放协作模式，任何 USTC 校友、在校生都可以通过以下几种方式参与。

## 贡献方式一览

| 类型 | 场景 | 怎么做 |
| --- | --- | --- |
| 汇入整本往届 PDF | 你手头有一份过去的《USTC 飞跃手册》《生存手册》PDF | 见下文 [PDF 汇入流程](#pdf-汇入流程) |
| 写单篇经验分享 | 想写一篇属于自己的故事（出国、保研、实习等） | 在对应章节新建 `.md` 文件提 PR |
| 补全某个章节 | 手册里已有章节为空，你想补齐 | 直接编辑对应 `.md`，提 PR |
| 修错别字、更新链接 | 发现错误或过时信息 | 提 Issue 或小 PR |
| 讨论结构 / 规范 | 对目录、模板、协作方式有建议 | 开 Issue 讨论 |

## PDF 汇入流程

**这是本手册内容填充的主要来源。** 如果你有往届 USTC 飞跃手册 / 生存手册的 PDF：

### 贡献者侧（你做的）

1. Fork 本仓库
2. 新建一个分支：`ingest/YYYY-<简短描述>`（例如 `ingest/2023-flyover`）
3. 把 PDF 放到 `_sources/` 目录下（此目录被 `.gitignore` 忽略，PDF 只用于本地提取，不进入仓库历史）
4. 开一个 Issue，标题 `[PDF 汇入] YYYY 届飞跃手册`，附上 PDF 的简介（页数、涉及专业、大致内容范围）
5. 在 Issue 里 at 维护者，或直接提 PR 请求协助

### 维护者/AI 辅助提取侧

1. 下载贡献者提供的 PDF（或贡献者在本地拆分后直接提 PR）
2. 按下列规则 **拆分并归档** 到对应章节：

   | PDF 中的原始内容类型 | 归档到 |
   | --- | --- |
   | 留学申请全流程经验 | `访谈集/为了留学而出国/` 下对应子文件（按主题合并） |
   | 单人留学 story / timeline | `附录/本科生专业介绍/` 或新建 `附录/出国故事/<学院>-<学号>.md` |
   | 保研全流程经验 | `访谈集/保研者说/` 下对应子文件 |
   | 考研经验 | `访谈集/考研.md` |
   | 找工作/实习经验 | `附录/本科生找实习经验分享/` |
   | 专业介绍（某届某专业视角） | `附录/本科生专业介绍/<学院>-<专业>-<届>.md` |
   | 转专业经验 | `附录/本科生转专业经验分享/<原专业>-to-<新专业>-<届>.md` |
   | 科研经历 | `附录/本科生科研经验分享/<方向>-<届>.md` |
   | 选课 / GPA / 考试心得 | `生存技巧/` 对应子章节 |
   | 对大学态度、反 PUA、做研究的兴趣等"价值观"类文字 | `立志篇/` 对应章节 |

3. 每篇切出来的文章，在开头加元信息块：

   ```markdown
   ---
   来源: YYYY 届 USTC 飞跃手册
   作者: 匿名 / <姓名或昵称>
   学院: <学院>
   专业: <专业>
   毕业去向: <去向>
   ---
   ```

4. 敏感信息（真实姓名、GT 分数、具体 offer 学校）按贡献者意愿做匿名化处理
5. 提交时 commit message 使用 `ingest(YYYY): <主题>` 前缀，便于追溯

### AI 协助提取的约定

如果你让 AI（比如 Claude）帮你做这件事，请在提示里明确：

- 不要改写原作者的观点和文风
- 可以纠正错别字、统一 markdown 格式
- 人名 / 学校 / 分数等按"匿名化"要求处理
- 不要把多篇文章揉成一篇"综合版"，而要保留独立身份 —— 读者可以对比多位作者的不同看法

## 写单篇经验分享的模板

在对应章节新建 `.md` 文件，推荐使用以下模板：

```markdown
---
作者: <姓名或昵称，可匿名>
毕业年份: 20XX
学院: <学院>
专业: <专业>
去向: <学校/公司/方向>
---

# <你的标题>

## 一些背景

（你的起点、主要挑战、为什么选这条路）

## 过程

（你做了什么，按时间线或模块）

## 经验与反思

（如果重来一次，你会怎么做）

## 建议

（给学弟学妹的一两条硬核建议）
```

## PR 规范

- 一个 PR 聚焦一件事（不要在同一个 PR 里既整理 PDF 又修别人的文字）
- commit message 请用中文或英文均可，但要说清"改了什么"
- 新文件名请使用中文或英文均可，但 **不要有空格**，用 `-` 或不加分隔

## 关于隐私

- 作者有权要求匿名
- 如果你引用了同学、老师的话，请先征得对方同意
- 不要在公开仓库里贴完整的学号、身份证号、真实姓名 + 分数段的组合
